# ArmExam Terminal

Kiosk-mode exam terminal for examination centers.

## Architecture

```
terminal/
├── backend/          Node.js + Express API (port 4000)
│   ├── server.js     PIN auth, sessions, results
│   ├── questions.json  Synced from main app
│   ├── exams.json      Synced from main app
│   └── data/
│       ├── sessions.json   All exam sessions (auto-created)
│       └── recordings/     Voice answer files (.webm)
└── app/              Electron + React (kiosk UI)
    ├── main.js       Electron: kiosk window, permission grants, shortcut locks
    ├── preload.js    IPC bridge (contextIsolation)
    └── src/
        ├── App.jsx          Theme switcher + screen router
        └── pages/
            ├── PinScreen.jsx    PIN entry screen
            ├── ExamScreen.jsx   Exam flow (timer, questions, voice, audio, video)
            └── ResultScreen.jsx Score / placement result
```

## Kiosk Mode Features

- **Electron kiosk**: `fullscreen + kiosk: true` — no title bar, no OS controls
- **Shortcuts blocked**: Alt+F4, F1–F12, Ctrl+W, Alt+Tab, Win+L, etc.
- **No popups**: `setWindowOpenHandler` denies all new windows
- **Navigation locked**: only localhost allowed, all external URLs blocked
- **Microphone auto-granted**: `setPermissionRequestHandler` allows `media/microphone`
- **Supervisor exit**: requires password (env `SUPERVISOR_PASSWORD`, default: `armexam-admin`)

## PIN System

PINs are JWT tokens signed with a shared secret.
- Generated by admin: `POST http://localhost:4000/api/pin/generate`
- Contains: `{ studentId, studentName, examId }`
- Default expiry: 30 days
- One-time use: once started, session is resumed on re-entry (not duplicated)

### Generate a PIN (curl example)
```bash
curl -X POST http://localhost:4000/api/pin/generate \
  -H "Content-Type: application/json" \
  -d '{"studentId": 1, "studentName": "Անի Հakobyan", "examId": 1}'
```

## Setup

### 1. Start Backend
```bash
cd terminal/backend
npm install
node server.js
# → running on http://localhost:4000
```

### 2. Start Terminal App (development)
```bash
cd terminal/app
npm install
npm run dev          # Vite dev server + Electron (windowed)
# or for pure UI testing:
npx vite             # browser at http://localhost:5173
```

### 3. Production (kiosk)
```bash
cd terminal/app
npm run build:ui     # build React → dist/
npm run dist         # package with electron-builder → .exe / .dmg / .AppImage
```

## Syncing Questions/Exams

When admin updates questions or exams, re-run from project root:
```bash
node --input-type=module -e "
import { QUESTIONS, EXAMS } from './src/data.js';
import { writeFileSync } from 'fs';
writeFileSync('./terminal/backend/questions.json', JSON.stringify(QUESTIONS, null, 2));
writeFileSync('./terminal/backend/exams.json', JSON.stringify(EXAMS, null, 2));
console.log('Synced');
"
```

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | /api/pin/generate | Generate PIN for student+exam |
| POST | /api/session/start | Validate PIN, start/resume session |
| POST | /api/session/answer | Save single answer |
| POST | /api/session/voice | Upload voice recording (multipart) |
| POST | /api/session/finish | Finalize exam, calculate score |
| GET  | /api/session/:id | Get session details |
| GET  | /api/sessions | List all sessions (admin) |
| GET  | /api/health | Health check |

## Question Types Supported

| Type | Input | Notes |
|------|-------|-------|
| single_choice | Radio buttons | Auto-scored |
| multi_choice / multi_select | Checkboxes | Auto-scored (exact match) |
| fill_blank | Text input | Auto-scored (case-insensitive) |
| writing | Textarea | Manual grading |
| audio | Audio player + choices | Replay limit enforced |
| voice | Mic recorder | Upload to backend, manual grading |
| (video support via videoSrc field) | Video player + choices | Replay limit enforced |

## Themes

Three themes available, switchable at runtime (top-right corner):
- **Dark** — default, dark charcoal background with gold accents
- **Neutral** — navy/indigo tones with blue-purple accents  
- **Light** — cream/off-white with brown-gold accents

Selection persists in localStorage between sessions.

## Environment Variables

| Variable | Default | Description |

| JWT_SECRET | `armexam-terminal-secret-2025` | PIN signing secret |
| SUPERVISOR_PASSWORD | `armexam-admin` | Password to exit kiosk |
